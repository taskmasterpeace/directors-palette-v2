# Seedance 2.0 — Phase 0 Findings

**Date:** 2026-04-19
**Engine:** `bytedance/seedance-2.0` on Replicate
**Test artist:** Country Fat (outlaw country rapper, banned words: `streetlights`, `flicker`, `neon`)
**Goal:** Validate engine viability for Music Lab "Produce Segment" feature before wiring into Shot Animator.

---

## Summary

**Seedance 2.0 is production-viable as a *text-to-video* engine only.** Any reference image
containing a realistic human face is rejected by ByteDance's input content filter (E005) before
inference starts. Adopt text-only prompts with DNA-driven identity description as the Phase 0
foundation.

---

Eight predictions run autonomously against the Replicate API using Country Fat's song slice

| 8 | Landscape-only crop (no face, background only) | **Generated 132s then E005 output-flag** | 132s |

Test 4 is the only reliable path. Test 8 is the diagnostic smoking gun.

## Two separate ByteDance filters

1. **Input filter** (fast-fail, 2–10s). Rejects any `reference_images` / `image` / `last_frame_image`
   containing a realistic face. Our own model-generated character sheets, clean headshots, and
   even Seedance's own generated frames all fail.
2. **Output filter** (slow, 120s+). Reviews the generated video post-inference. Can reject content
   that passed the input filter — test 8 had no face anywhere and still flagged after full generation.

## What worked (`pred4.mp4`)

- **Model:** `bytedance/seedance-2.0`
- **Inputs:** `prompt` only. No `reference_images`, no `image`, no `last_frame_image`, no audio ref.
- **Resolution:** 480p (cheapest tier; 720p also available; 1080p does **not** exist for Seedance 2.0).
- **Duration:** 15s (engine max).
- **Result:** 5.1MB MP4. Cowboy hat, freckles, hoodie, Texas highway, golden hour, pickup truck,
  mesquite scrub, cracked asphalt — all captured purely from DNA-driven text.

This proves the DNA lexicon (`look`, `lexicon.vibeWords`, filtered against `lexicon.bannedWords`)
is sufficient for identity-preserving generation without a face reference.

## Schema notes (verified from Replicate)

```
resolution:      "480p" | "720p"           # NO 1080p
aspect_ratio:    "16:9" | "9:16" | "1:1" | "4:3" | "3:4" | "21:9" | "9:21" | "adaptive"
duration:        integer, -1..15           # -1 = "intelligent" auto-duration
generate_audio:  boolean                   # enable native lip-sync
reference_images: up to 9                  # MUTEX with image + last_frame_image
reference_videos: up to 3, 15s total       # motion/camera transfer
reference_audios: up to 3, 15s total       # requires at least 1 image or video ref
image, last_frame_image:                   # single-ref mode (mutex with reference_images)
```

Bracket-tag syntax in prompts: `[Image1]`, `[Audio1]`, `[Video1]`. Lyrics in **double quotes**
trigger native Seedance lip-sync when `generate_audio=true`.

## Implications for the app

1. **Character sheet as Seedance ref is dead.** The app's existing identity-preservation pattern
   (feed sheet/portrait as reference) will 100% fail with Seedance 2.0.
2. **DNA becomes the prompt engine.** Artist `look` + `lexicon.vibeWords` (filtered against
   `bannedWords`) must be synthesized into rich identity-preserving prompts by the LLM before
   calling Seedance.
3. **Filter-safe refs only.** Non-face object stills (locations, wardrobe close-ups, props) and
   motion video refs appear viable for the input filter. But the output filter can still reject.
4. **Lip-sync still works.** `generate_audio=true` + double-quoted lyrics is supported from
   text-only mode — no face ref needed.
5. **Identity drift across shots** is the main creative trade-off. Mitigation: lock director
   fingerprint, wardrobe lexicon, and location lexicon per music-video segment so continuity is
   description-driven rather than pixel-driven.
