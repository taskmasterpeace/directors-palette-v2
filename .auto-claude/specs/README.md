# Specs Directory

This directory contains task specifications generated by Auto-Claude.

## Structure

Each spec gets its own numbered folder:
```
specs/
├── 001-feature-name/
│   ├── spec.md                  # Full specification
│   ├── implementation_plan.json # Subtasks and phases
│   ├── context.json             # File context
│   └── requirements.json        # Original requirements
├── 002-another-feature/
│   └── ...
```

## Spec File Format

### spec.md
Contains the full task specification with sections:
- Overview
- Workflow Type (feature/refactor/investigation/migration/simple)
- Task Scope
- Service Context
- Files to Modify
- Files to Reference
- Patterns to Follow
- Requirements
- Implementation Notes
- Development Environment
- Success Criteria
- QA Acceptance Criteria

### implementation_plan.json
Defines the execution plan:
```json
{
  "feature": "Feature Name",
  "workflow_type": "feature",
  "phases": [
    {
      "id": "phase-1",
      "name": "Phase Name",
      "subtasks": [
        {
          "id": "subtask-1-1",
          "description": "What to do",
          "files_to_modify": ["path/to/file.ts"],
          "verification": {
            "type": "command",
            "command": "npm run build",
            "expected": "Build succeeds"
          },
          "status": "pending"
        }
      ]
    }
  ]
}
```

## Creating a New Spec

1. Create a numbered folder (e.g., `003-my-feature/`)
2. Add the spec files
3. Run Auto-Claude to execute

## Verification Types

- `command`: Run a CLI command
- `api`: Test an HTTP endpoint
- `browser`: Check UI rendering
- `e2e`: Run E2E test
- `manual`: Requires human verification
